# Contributing to FireForm

Thank you for your interest in contributing! FireForm is a Digital Public Good (DPG) designed to help emergency responders.

## 🌈 Code of Conduct

All contributors are expected to uphold the [FireForm Code of Conduct](CODE_OF_CONDUCT.md).

## 🚀 How Can I Contribute?

### Reporting Bugs

Check the [issues list](https://github.com/fireform-core/FireForm/issues) before reporting. If it's a new bug, please include:
- A clear title.
- Steps to reproduce.
- Expected vs. actual behavior.

### Suggesting Enhancements

Open an issue with:
- A clear title.
- A detailed description of the feature.
- Rationale for why it's useful.

### Pull Requests

1.  Fork the repo and create your branch from `main`.
2.  Add tests if you've added new features.
3.  Ensure the test suite passes.
4.  Make sure your code lints.
5.  Open the pull request!

## 🛠️ Local Development Setup

FireForm uses Docker for development. See the [[DOCKER]] page for more details.
